# PI AGENT Guide

Git is read-only.

Prefer simple over complex and follow the rule of three for
refactoring/abstracting. For helpers, let's not needlessly recycle code and use
shared helpers when we can.

Don't make documentation too verbose. ~80ch width (save for tables, which can
be long) and ~200-250 lines is good before considering if it can be reworded or
split up.
